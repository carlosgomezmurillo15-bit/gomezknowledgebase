# Assignment 02: Analyzing an Organizing System

  

## Organizing System Selected: Steam Store

  

For this assignment, I chose the **Steam Store** as the public digital collection to analyze. Steam is an online platform where users can browse, purchase, review, join/explore game communities, and organize video games. I chose Steam because it has a large amount of content, and its organizing system helps users find games based on genre, price, popularity, system compatibility, tags, and other user preferences.

  

---

  

## Primary Resources Being Organized

  

The primary resources being organized on Steam are **video games and game-related content**. This includes full games, downloadable content, demos, soundtracks, software, bundles, and community created content. Each game has its own page that includes a title, description, screenshots, trailers, system requirements, reviews, release date, developer, publisher, price, communities, and tags.

  

For example, a game like *Elden Ring* is not just listed as a product. It is connected to genres such as action RPG, tags such as open world and fantasy, user reviews, similar games, and further developer information making each resource easier to understand and compare before a user decides to buy or download it.

  

---

  

## Primary Interactions Supported

  

Steam supports several important interactions. The first is **searching**, where users can type the name of a specific game or keyword into the search bar. It also supports **browsing**, because users can explore categories like New & Trending, Top Sellers, Specials, Free to Play, or specific genres.

  

Another major interaction is **filtering and comparison**. Users can narrow results by price, genre, user reviews, release date, operating system, number of players, and tags. This is useful because Steam has thousands of games, and a simple alphabetical list would not be helpful at all. Steam also supports **decision-making** through reviews, ratings, trailers, screenshots, and recommendation sections.

  

---

  

## Classification System

  

Steam mainly uses a **faceted classification system**. Instead of forcing every game into only one fixed category, Steam lets users combine multiple filters. For example, a user can search for games that are action, multiplayer, under a certain price, highly reviewed, and compatible with Windows. This makes the system flexible because different users care about different qualities.

  

Steam also uses some hierarchical organization through broad categories like genres and store sections. However, the faceted system is more important because it allows users to narrow down results in a personalized way. This is effective for Steam's audience because gamers often search based on several needs at once, such as price, genre, reviews, required operating systems and gameplay style.

  

---

  

## Relationships Between Resources

  

Steam represents relationships between resources through **tags, recommendations, bundles, developer pages, user reviews, and "similar games" sections**. Tags are especially important because they connect games across different categories. For example, two games may both be tagged as survival, open world, or co-op even if they are made by different companies.

  

Steam also links games to related downloadable content, sequels, franchises, and developer or publisher pages. User reviews create another relationship by connecting players' experiences to the resource. Recommendation sections help users discover games similar to ones they already viewed or purchased.

  

Overall, Steam is a very effective organizing system as it supports both direct searching and open ended browsing. Its faceted classification system works well for a large digital game collection because it gives users multiple ways to narrow, compare, and even discover new resources.
